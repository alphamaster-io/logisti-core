# openspec/

This directory drives **spec-driven development** for LogistiCore.

| You want to …                                          | Read                              |
| ------------------------------------------------------ | --------------------------------- |
| Understand what LogistiCore is and the non-negotiables | `project.md`                      |
| Learn the proposal → implement → archive workflow      | `AGENTS.md`                       |
| See what the system currently does (the "is")          | `specs/<capability>/spec.md`      |
| See what's being proposed (the "will be")              | `changes/<change-id>/proposal.md` |
| See an archived decision                               | `changes/archive/<change-id>/`    |

The OpenSpec CLI (`npx openspec`) validates structure, but the format is plain markdown — manual review works.
